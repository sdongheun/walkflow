# Walkflow MVP 개발 기획 및 세팅 가이드

Walkflow 앱의 MVP 사양을 문서화하고, 앞으로 필요한 개발 항목 및 단계별 구현 목표를 정리합니다.

## User Review Required

> [!IMPORTANT]
> 정리된 MVP 요구사항(`mvp_spec.md`)과 전반적인 개발 태스크(`task.md`)를 확인해 주세요.
> 앱의 방향성과 주요 기능(건강앱 연동, 경로 및 신호 탐색, 시간 알림)이 맞는지 검토가 필요합니다.
> 추가로 필요한 기능이나 수정하고 싶은 부분이 있다면 말씀해 주세요!

## Proposed Changes

현재 프로젝트 소스 코드에는 기능적 변경사항이 없으며, 프로젝트의 개발 목표를 문서화하여(Artifacts) 저장했습니다.
향후 진행될 변경 사항은 다음과 같습니다.

### [React Native / Expo App 구성]

#### [NEW] `src/screens/HomeScreen.tsx`

- 사용자 현재 위치 기반 지도 렌더링 (Kakao/Naver Map 연동)
- 목적지 검색 및 "최적 출발 시간 알림" 메시지 UI 구현

#### [NEW] `src/store/useUserStore.ts`

- Zustand를 활용하여 사용자의 보행 속도 데이터(기본값 1.2m/s 및 건강앱 실시간 속도) 전역 상태 관리

#### [NEW] `src/utils/healthData.ts`

- 기기 내 건강 앱 데이터 접근 권한 요청 및 최근 1개월 평균 보행 속도 추출 로직

#### [NEW] `src/utils/trafficApi.ts`

- 공공데이터포털 API 연동하여 근처 횡단보도의 신호 주기(초록불/빨간불)를 실시간으로 가져오는 로직

#### [NEW] `src/utils/timeCalculator.ts`

- 사용자 위치부터 횡단보도까지의 거리 / 사용자 보행 속도를 계산하여 도착 예정 시간 산출
- 신호등 주기와 매칭하여 최적 출발 시간 계산

## Verification Plan

### Automated Tests

- 현재 MVP 단계에서는 빠르게 기능을 검증하는 것이 목적이므로, 복잡한 자동화 테스트보다는 각 API 연동 및 상태 관리가 정상 동작하는지 로그를 통해 확인합니다.

### Manual Verification

1. **보행 속도 검증**: 앱 실행 시 건강앱 권한을 요청하고, 허용 시 본인의 평균 보행 속도가 저장되는지 확인. 거부 시 기본값(1.2m/s)이 저장되는지 확인.
2. **지도 및 경로 검증**: 화면에서 출발지와 목적지를 설정했을 때, 중간에 거쳐야 할 가장 알맞은 횡단보도 위치 한 곳을 앱이 제대로 잡는지 확인.
3. **신호 데이터 및 최적 시간 검증**: 산출된 최적 출발 시간이 실제로 해당 횡단보도 신호등의 녹색불 타이밍과 맞춰지는지 (길거리에서 직접 걷거나 개발자 도구로 가상 위치를 이동시켜서) 실제 타이머를 보며 테스트.
